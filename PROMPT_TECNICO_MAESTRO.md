# Prompt Técnico Maestro (ASPECT)

## Sistema: MALAMBO TATTOO SLOT

### A — Action (Acción)
Redacta las instrucciones completas para que un programador construya el sistema **"MALAMBO TATTOO SLOT"** con precisión técnica, cumplimiento normativo argentino 2026 y transparencia total, incluyendo frontend, backend, lógica y requisitos legales hard-coded.

### S — Subject (Sujeto)
Interfaz de gamificación que simula una **máquina de premios (slot)** para el estudio **MALAMBO TATTOO**, con apariencia vintage tipo Las Vegas, marca en neón blanco y recompensas de tatuajes.

### P — Purpose (Propósito)
Asegurar una experiencia visual atractiva y un sistema de azar **100% certificado** bajo **GLI-19 v3.0**, con emisión de certificado público por premio, recolección segura de datos de contacto, y cumplimiento estricto de la normativa argentina vigente (2026).

### E — Environment (Entorno)
Aplicación web con frontend (UI + animaciones) y backend (lógica de azar, certificación, datos de contacto). Debe estar preparada para integrarse con bases legales registradas ante **LOTBA S.E.** para participantes en **CABA**.

### C — Constraints (Restricciones)
- **No usar frutas ni números** en los rodillos.
- **Leyendas legales obligatorias** visibles y destacadas.
- **+18** y advertencia de salud deben ocupar **al menos 10%** del anuncio o interfaz.
- Implementar **URL pública de certificado** por cada premio.
- **Aleatoriedad certificada GLI-19 v3.0**.

### T — Tone (Tono)
Técnico, claro, exhaustivo y orientado a implementación; con lenguaje legal preciso y verificable.

## Prompt Técnico Maestro (Listo para copiar y usar)

**Rol:** Eres un desarrollador senior responsable de construir el sistema "MALAMBO TATTOO SLOT" para el estudio MALAMBO TATTOO, cumpliendo estrictamente con normativa argentina 2026 y transparencia total. Debes entregar una solución implementable con precisión técnica.

**Objetivo general:** Construir una interfaz de gamificación tipo máquina tragamonedas, con estética vintage Las Vegas, premios específicos de tatuajes, animaciones 3D y lógica de azar certificada GLI-19 v3.0, incluyendo generación de certificado público por premio, captación segura de datos de contacto y cumplimiento de requisitos legales obligatorios.

### 1) Especificaciones de Diseño (Frontend)
1. **Identidad visual:**
   - Estética de **máquina vintage de Las Vegas**.
   - El cartel superior (marquee) debe mostrar el **logotipo circular de MALAMBO TATTOO** ("M" ornamentada) en **neón blanco**.
2. **Rodillos (Reels):**
   - Prohibido usar frutas o números.
   - Símbolos permitidos (y únicos):
     - "Tatuaje Pequeño"
     - "2x1"
     - "3x1"
     - "Tattoo 15cm"
     - Jackpot: "Sesión Completa"
3. **Animación de victoria:**
   - Al ganar, ejecutar **lluvia de monedas de oro en 3D**.
   - Monedas con **logo de la marca acuñado en relieve en ambas caras**.

### 2) Lógica y Transparencia (Backend)
1. **Algoritmo de azar:**
   - Implementar selección aleatoria automatizada conforme a **GLI-19 v3.0**.
   - El sistema debe garantizar resultados **100% azarosos e inalterables**.
2. **Certificación pública:**
   - Al finalizar cada giro ganador, generar un **Certificado de Validez público**.
   - El certificado debe estar disponible en una **URL única**.
   - Debe incluir **fecha, hora y premio adjudicado**.
3. **Gestión de datos:**
   - Implementar flujo "**Reclama tu premio**".
   - Recolectar **datos de contacto** de forma **segura y conforme a normativa de protección de datos**.

### 3) Requisitos Legales Obligatorios (Hard-Coded)
Incluir de forma **visible, destacada y persistente**:
1. **Leyenda de compra:**
   - "**Sin obligación de compra**" (Decreto 274/2019).
2. **Advertencias de salud y edad:**
   - "**EL JUGAR COMPULSIVAMENTE ES PERJUDICIAL PARA LA SALUD**".
   - Símbolo **+18**.
   - Ambas advertencias deben ocupar **mínimo 10% del anuncio o interfaz**.
3. **Jurisdicción:**
   - Advertir que la promoción **puede no ser válida** en provincias con regulaciones fiscales específicas (ej. **Córdoba o Mendoza**).
4. **Autorización:**
   - El sistema debe estar preparado para **vincularse a bases legales registradas ante LOTBA S.E.** (CABA).

### 4) Flujo de razonamiento (Logic Layer)
Antes de cada ejecución del giro, el sistema debe **razonar paso a paso de forma interna** y ejecutar en este orden:
1. Validar mayoría de edad del usuario (**+18**).
2. Consultar bases y condiciones vigentes.
3. Ejecutar el algoritmo aleatorio certificado.
4. Emitir certificado de validez inalterable en caso de premio.

> Nota: El razonamiento interno no debe exponerse al usuario final; solo se muestran resultados y mensajes de estado.

### 5) Entregables mínimos
- UI completa con estética especificada.
- Algoritmo de azar GLI-19 v3.0 documentado e integrado.
- Generación de certificado público con URL única.
- Formulario seguro de reclamo de premio.
- Leyendas legales hard-coded con visibilidad mínima del 10%.
